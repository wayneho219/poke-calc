# i18n — Trilingual UI Support Design

**Date:** 2026-04-22  
**Project:** poke-calc (`/Users/wayneho/poke-calc`)  
**Scope:** Add Traditional Chinese / English / Japanese UI language switching to the Streamlit app

---

## Background

The existing app supports trilingual Pokémon *name* search (input and result display show all three languages). However, all UI chrome — tab labels, headers, input labels, buttons, error messages, stat names, type names — is hardcoded in Traditional Chinese. This design adds full UI i18n so any user can operate the app in their preferred language.

---

## Goals

- All visible UI strings switchable between 繁體中文 / English / 日本語
- Language persists across page reloads (via URL query param `?lang=xx`)
- First visit auto-detects browser language via `Accept-Language` header
- Pokémon type names (dragon, ground, fire…) translated per language
- No new pip dependencies

---

## Non-Goals

- Machine translation of arbitrary user input
- More than three languages
- Locale-specific formatting (dates, numbers)
- Domain/application layer changes (i18n is UI-only)

## Components

### `Translator` class (`shared/i18n/translator.py`)

```python
class Translator:
    def __init__(self, lang: str) -> None:
        path = I18N_DIR / f"{lang}.json"
        self._data = json.loads(path.read_text(encoding="utf-8"))

    def __call__(self, key: str) -> str:
        """Return translated string. Falls back to key itself if missing."""
        return self._data.get(key, key)

    def type_name(self, type_en: str) -> str:
        """Translate a Pokémon type name (e.g. 'dragon' → '龍')."""
        return self._data.get("types", {}).get(type_en, type_en)
```

Fallback behaviour: if a key is missing in a JSON file, `t("key")` returns `"key"` — the app degrades gracefully rather than crashing.

## Language Detection Flow (`app.py`)

```
Page load
  │
  ├─ st.query_params has "lang"? ──yes──► use it (validate: zh/en/ja, else zh)
  │
  └─ no ──► read st.context.headers["Accept-Language"]
              │
              ├─ starts with "zh" ──► "zh"
              ├─ starts with "ja" ──► "ja"
              ├─ other / unrecognised ──► "zh"  (default to Traditional Chinese)
              └─ AttributeError       ──► "zh"  (Streamlit < 1.37 fallback)
```

The header parsing logic is extracted into a pure function `parse_accept_language(header: str) -> str` (in `shared/i18n/translator.py`) so it can be unit tested without Streamlit. `detect_lang()` in `app.py` calls this function after reading the header.

### Sidebar language selector

```python
with st.sidebar:
    chosen = st.selectbox(
        "🌐 Language",
        options=["zh", "en", "ja"],
        format_func=lambda l: {"zh": "繁體中文", "en": "English", "ja": "日本語"}[l],
        index=["zh", "en", "ja"].index(lang),
    )
    if chosen != lang:
        st.query_params["lang"] = chosen
        st.rerun()
```

Changing language updates `?lang=xx` in the URL and triggers a Streamlit re-run. The URL persists on reload.

## Nature Selectbox Localisation

The hardcoded English nature lists in the Speed and Survival tabs are replaced with localised options read from the JSON. Each JSON file contains `"nature_other"` for the freeform fallback option. The selectbox values remain as English nature names internally (since `NatureRegistry.get_by_name` accepts all three languages), but the display label shown to the user matches the active language.

For simplicity, the preset nature options (Timid, Jolly, etc. for Speed; Bold, Impish, etc. for Survival) are kept as English keys in the JSON — they are universally recognisable to competitive Pokémon players. Only the "Other..." option is localised.
